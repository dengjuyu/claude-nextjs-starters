현재 변경사항을 분석하고 정교한 커밋을 생성해주세요.

## 프롬포트
git status로 변경사항을 확인한 후, 다음을 수행:
1. 수정된 파일들을 분석하여 작업 내용 파악
2. 커밋 컨벤션에 맞는 한국어 커밋 메시지 작성
3. 변경사항을 아이템 리스트로 정리
4. git add + git commit으로 직접 커밋 생성

## 커밋 메시지 형식
```
<type>: <작업 내용 요약>

- 작업 내용 1
- 작업 내용 2
- 작업 내용 3
```

### Type 규칙
- `feat:` - 새로운 기능 추가
- `fix:` - 버그 수정
- `refactor:` - 코드 정리/구조 변경 (기능 변화 없음)
- `docs:` - 문서 추가/수정
- `style:` - 스타일/포맷 변경 (CSS, 띄어쓰기 등)
- `test:` - 테스트 추가/수정
- `chore:` - 빌드, 의존성, 설정 변경

### 작성 규칙
1. **요약(Subject)**: 50자 이내, 마침표 없음, 명령형 현재시제
2. **본문(Body)**: 한 줄 띄고 시작, 72자 이내로 줄 바꿈
3. **언어**: 모두 한국어로 작성
4. **범위(Scope, 선택사항)**: 큰 변경시만 추가 `feat(auth): 로그인 기능`
5. **논리적 단위**: 한 번에 하나의 의미있는 변경만 커밋
6. **파일 제외**: node_modules, .next, .DS_Store 등 불필요한 파일 제외
